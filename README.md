# 성빈이 전용 생일 페이지 🌹

QR 카드로 열어보는 모바일 전용 생일 축하 정적 웹페이지입니다. 프레임워크나 외부 라이브러리 없이 HTML, CSS, JavaScript만 사용하므로 별도 빌드 없이 Vercel에 배포할 수 있습니다.

## 파일 구조

```text
.
├── index.html        # 화면 구조와 생일 메시지
├── style.css         # 모바일 카드 UI와 애니메이션
├── script.js         # 단계 전환, 버튼 장난, burst 효과
├── assets/
│   └── README.md     # 선택 이미지 파일 안내
├── robots.txt        # 검색 엔진 수집 차단
├── vercel.json       # Vercel 헤더와 정적 배포 설정
└── README.md
```

## 로컬 실행

파일을 직접 열어도 동작하지만, 브라우저의 정적 파일 처리 차이를 피하려면 프로젝트 폴더에서 간단한 서버를 실행하는 방식을 권장합니다.

Python이 있다면:

```bash
python -m http.server 8000
```

브라우저에서 `http://localhost:8000`을 여세요. Node.js가 있다면 `npx serve .`도 사용할 수 있습니다.

## UX 흐름

1. 첫 화면에서 생일 축하를 받을지 묻습니다.
2. `아니요` 버튼은 마우스, 터치 또는 키보드 입력에 반응해 넓어진 영역 안에서 잔상을 남기며 도망갑니다. 세 번 시도하면 거절 도전 미니게임이 열립니다.
3. `사랑 피하기`에서 `성빈` 이름표가 달린 여자 사람 이모지 캐릭터를 좌우로 움직여 하트와 장미를 피합니다. 30초를 버티면 거절 성공이지만 3초부터 난이도가 급격히 올라갑니다.
4. 모든 선택지가 긍정인 화면에서 아무 버튼이나 누르면 장미·하트·케이크가 터집니다.
5. 마지막 확인에서 `취소`를 누를수록 버튼 자체가 입장 문구로 바뀌며, 마지막에는 생일 화면으로 입장합니다. `입장하기`는 즉시 입장합니다.
6. 메인 화면에는 생일 축하, 붉은 장미 선정 이유, 6월 9일의 기억, 마지막 편지가 순서대로 표시됩니다.
7. 엔딩 버튼을 누르면 아이콘이 다시 터지고 축하 메시지가 나타납니다.

PC에서 열어도 페이지 전체가 최대 430px 너비의 모바일 카드로 가운데 정렬됩니다. 작은 화면과 iOS 안전 영역을 고려했고, `prefers-reduced-motion` 사용자는 움직임과 생성되는 아이콘 수가 줄어듭니다.

## 이미지 교체하기

`assets/` 폴더에 아래 파일을 추가하면 burst 효과가 자동으로 해당 이미지를 사용합니다.

```text
assets/rose.png
assets/heart.png
assets/cake.png
assets/custom-1.png
```

이미지가 없거나 로드에 실패하면 각각 🌹, ❤️, 🎂, 💖 이모지가 표시되므로 이미지 추가 전에도 완전히 동작합니다. 로딩을 가볍게 유지하기 위해 이 네 파일만 최소한으로 확인합니다.

## Vercel 배포

1. 이 저장소를 Vercel에 Import합니다.
2. Framework Preset은 `Other`로 둡니다.
3. Build Command는 비워두고, Output Directory는 `.` 또는 기본값을 사용합니다.
4. Deploy를 누릅니다.

배포가 끝나면 발급된 Vercel URL을 QR 코드 생성기에 넣어 카드에 인쇄하면 됩니다. 나중에 커스텀 도메인을 연결했다면 QR에는 최종 도메인을 사용하는 편이 좋습니다.

> `robots.txt`, HTML 메타 태그, Vercel의 `X-Robots-Tag` 헤더로 검색 노출을 막도록 설정했습니다. 다만 URL을 아는 사람의 접근 자체를 막는 인증 기능은 아닙니다.

## 기술 메모

- 외부 라이브러리, 웹폰트, 배경음악 없음
- 실제 `button` 요소와 `aria-live` 안내 영역 사용
- iOS Safari 및 Android Chrome에서 사용할 수 있는 표준 DOM API 사용
- burst 아이템은 한 번에 최대 20개이며 약 1~1.5초 뒤 DOM에서 제거
